[#configuration]
== Service Binding and Configuration

On the Deployment Detail page, click on the Pods tab to find a list of Pod resources that are attached to this Deployment

image::deploy_1.png[]

Select one of the Pods

image::deploy_2.png[]

Then, open the `Terminal` tab from the Pod Detail page

image::pod_terminal.png[]

1. Check the backend database for records. The result should be empty:
+
[.console-input]
[source,bash,subs="+macros,+attributes"]
----
curl localhost:8080/ws/data/all
----

2. Populate the backend database with the National Parks dataset:
+
[.console-input]
[source,bash,subs="+macros,+attributes"]
----
curl localhost:8080/ws/data/load
----

3. List registered gateway backends. The only result should be the gateway's built-in "Red Hat Summit POI" dataset:
+
[.console-input]
[source,bash,subs="+macros,+attributes"]
----
curl poi-gateway-app:8080/backend/list
----

4. Register the new backend component with the gateway component:
+
[.console-input]
[source,bash,subs="+macros,+attributes"]
----
curl poi-gateway-app:8080/backend/register/poi-backend?endpoint=http://poi-backend-app:8080
----

Check the map for results!

image::map_3.png[]
